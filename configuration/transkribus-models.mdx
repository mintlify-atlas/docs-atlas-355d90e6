[Transkribus](https://readcoop.eu/transkribus/) is a platform for handwritten text recognition (HTR) developed by the READ-COOP cooperative. It specializes in historical manuscripts and documents where conventional OCR engines — which are optimized for printed text — perform poorly. The Wikisource extension integrates with Transkribus via the OCR service to allow transcription of handwritten source material.

## Model types

Transkribus uses two types of models:

- **HTR models** recognize the text content of a handwritten or historical document. Each model is trained on a specific script, language, or time period. You must select at least one HTR model per wiki.
- **Line detection models** identify the individual lines of text on a page before the HTR model reads them. Line detection is optional — if omitted, the OCR service uses its default line segmentation. A line model is specified by its numeric model ID.

## Configuration structure

Use `$wgWikisourceTranskribusModels` in `LocalSettings.php` to configure models on a per-wiki basis. The top-level keys are wiki database names (the value of `$wgDBname` for each wiki). Each entry contains an `htr` array of model name strings and an optional `line` integer model ID.

```json
{
  "<wiki-db-name>": {
    "htr": ["<model-name>", "..."],
    "line": 12345
  }
}
```

Models listed in the `htr` array appear as selectable options in the OCR settings panel when a user chooses the Transkribus engine. The `line` value, when present, is automatically sent as the `line_id` parameter to the OCR API request.

## Fetching and caching

Available Transkribus languages and models are fetched from the OCR service at:

```
GET {$wgWikisourceOcrUrl}/api/available_langs?engine=transkribus
```

This request is made server-side when the `ext.wikisource.OCR` resource module is built. Results are stored in the MediaWiki WAN object cache under a global key and are valid for **one day** (`TTL_DAY`), with a stale serving window of **one week** (`TTL_WEEK`). If the OCR service is unreachable, the extension returns an empty array rather than failing hard.

## Adding your own wiki

To enable Transkribus for a wiki not listed above, add an entry to `$wgWikisourceTranskribusModels` in `LocalSettings.php`. Use the wiki's database name as the key and provide at least one HTR model name.

```php
$wgWikisourceTranskribusModels = [
    // Add your wiki alongside the defaults, or replace this array entirely
    'mywikisource' => [
        'htr' => [ 'my-model-name' ],
        'line' => 99999, // optional line detection model ID
    ],
];
```

<Note>
  Model names must match identifiers recognized by the Transkribus API. Contact your OCR service operator or check the Transkribus platform for available model names for your language and script.
</Note>
